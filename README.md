# TangerineAuction

### Стек: 
- c# (.net 10)
- React
- Postgres
- Rabbit mq
- Redis
- Keycloak

### Проекты:
- TangerineAuction.Web - веб апи + фронт
- TangerineGenerator.Service - микросервис для генерации мандаринок

### TangerineGenerator.Service
Генерация мандаринок каждые 5 минут (настраиваемое). Генерирует редкость, название, цену и картинку. Название, цена и картинка зависят от редкости. Картинка может содержать следующие элементы:
- Задний фон,
- Звёзды,
- Тень,
- Мандарин,
- Точки на мандарине,
- Блик,
- Ореол,
- Листик.

### Интеграция:
#### Mass transit:
- Делает пабиш события, что мандаринка сгенерирована, и передаёт её данные в основное приложение;
- Содержит Request для принудительного запуска генерации мандаринки.
#### GRPC: 
- Получить версию сервиса;
- Удалить картинку из папки.

### TangerineAuction.Web:
- Работа с сущностями (аукционы, мандарины, ставки);
- Работа с Keycloak (получение токена для confidential-client, создание системного пользователя);
- Отправка сообщений на почту.

#### Интеграция:
- Содержит событие, что мандаринка сгенерирована.
- Делает паблиш Request-а для принудительного запуска генерации мандаринки.

### Алгоритм работы:
1. TangerineGenerator.Service генерирует мандаринку (каждые 5 минут) и паблишит запрос.
2. TangerineAuction.Web обрабатывает запрос:
- создаёт аукцион;
- создаёт ставку равную изначальной стоимости мандаринки, автор ставки - системный пользователь;
- всё сохраняет в БД.

Раз в день TangerineAuction.Web проходит активным аукционам и проверяет кол-во ставок. Если ставок > 1, то отправляет чек победителю на почту и переводит аукцион в статус завершённого.
Если ставка только одна, то удаляет аукцион, мандаринку, ставки и фото мандаринки (по GRPC вызывает соответствующий метод у TangerineGenerator.Service).

Если пользователь перебил ставку, то отправляет письмо на почту пользователю, у которого перебили ставку.
